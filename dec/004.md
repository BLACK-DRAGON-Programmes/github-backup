# Decision 004 — Notification System (Windows Toasts)

## Date

2026-05-25

## Status

Accepted

## Context

The original specification (Section 9 — Non-Requirements) listed "No notification system (email, webhook, etc.)" as explicitly out of scope. A request was made to amend the specification to add a notification system using Windows native toast notifications.

## Decision

Add a Windows toast notification system. Toasts fire for **every type of event** — not just errors. This includes:

- Per-repo backup success
- Per-repo backup failure (any type: 404, network error, disk full, etc.)
- Internet connectivity check result (no internet detected)
- Rate limit hit
- Corrupt `.env` file detected
- Cycle start and cycle completion summary
- Any other runtime event

## Behavior

- **Trigger:** Individual toast per repo, plus additional toasts for non-repo events (connectivity, config errors, etc.)
- **Content:** All available information — repo name (if applicable), action, status (success/failure), timestamp, error details (if applicable)
- **Duration:** Standard Windows toast auto-dismiss behavior (no persistent/to-click-dismiss)
- **Implementation:** Win32 `ToastNotification` API or `Shell_NotifyIcon` with balloon tooltips (Windows 10 native toast infrastructure via `windows.ui.notifications` COM interface or simpler `MessageBoxTimeout` / `Shell_NotifyIconW` approach)

## Rationale

The script runs as a background scheduled task with no console window. Without notifications, there is zero visibility into whether the script is functioning, encountering errors, or silently failing. Toasts provide passive, non-intrusive feedback — they appear in the Action Center even if dismissed, so they can be reviewed later.

## Impact on File Plan

New source files required:
- `src/notify.c` — Toast notification implementation
- `src/notify.h` — Notification function prototypes and constants

Notification constants added to `src/constants.h`.

## Spec Changes

- Section 9: "No notification system" removed from out-of-scope list
- New section added: Windows toast notification requirements (trigger conditions, content, duration)
- Section 7 (Error Handling): each error type now includes "fire toast notification" as part of the error response
